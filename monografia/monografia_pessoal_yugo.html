Desafios e frustrações econtrados


Um dos objetivos do grupo ao realizar este trabalho foi a de adquirir experiência de desenvolvimento de sistemas, e um dos grandes desafios encontrados por muitos desenvolvedores é escolher as ferramentas que serão utilizadas no sistema. E para o autor desta monografia esse também foi um grande desafio.

Inicialmente a pedido do autor o grupo aceitou adotar o uso da IDE JDeveloper e o servidor de aplicações OC4J para a implementação do sistema. Mas as aplicações geradas por esta IDE se provou extremamente difícil de ser configurado para outros servidores como o Tomcat e o JBoss, e como a idéia do professor era tornar este sistema um software livre e com código aberto, este simplesmente não poderia somente funcionar na implementação particular da Oracle de servidor de aplicação (OC4J).

Após inúmeras frutrações tentando aumentar a portabilidade do sistema com o JDeveloper o grupo chegou à conclusão que seria melhor abandonar o JDeveloper. Para evitar mais perda de tempo com ferramenta mal escolhida, o grupo resolveu gastar um tempo pesquisando qual seria o melhor ambiente de desenvolvimento, e o ambiente resultante está detalhado na monografia em grupo.

No grupo de componentes que o autor indicou para o grupo utilizar, não há suporte direto para tabelas com colunas dinâmicas ainda. E isto acabou gerando um grande desafio para o autor de criar colunas via código Java manipulando diretamente os componentes que estão em uma página. Apesar de essa possibilidade ser uma funcionalidade do arcabouço, quase não há exemplos de situações mais complexas como há que o autor encontrou de gerar colunas dinâmicas com suporte a arrastar e derrubar objetos gerando algum comportamento. Mesmo com a falta de exemplos através de tentativa e erro o autor conseguiu um bom compreendimento das funções mais complexas do arcabouço e com isso a situação foi contornada.


Trabalho e o curso


Laboratório de Programação 2, Programação Orientada a Objetos e Tópicos Avançados de Programação Orientada a Objetos: Onde foi primeiro introduzido a noção de orientação a objetos. Para desenvolver sistemas grandes este é o conceito mais importante para poder implementar em grupo eficientemente. Há outras maneiras além de orientação a objetos, mas na opinião do autor esta é a forma mais eficiente e que produz o código mais legível para futuras mudanças.

Estrutura de dados: Primeira noção de como trabalhar com estruturas além de listas, como por exemplo mapas e árvores. Muito importante para saber decidir quando utilizar qual tipo de estrutura.

Introdução à Computação, Princípios de Desenvolvimento de Algoritmos e Análise de Algoritmos: Primeira noção de buscas em listas e eficiência de código. Bastante utilizados para saber analisar quando utilizar uma busca e evitar códigos que podem diminuir muito a performance de um sistema.

Interação Humano Computador: A única matéria que revela a importância de uma interface boa. A usabilidade e a aceitabilidade da interface do sistema são as principais causas de fracassos ou desuso de um sistema, e saber princípios de uma boa interface pode agregar muito valor a um sistema sendo projetado.

Sistemas de Bancos de Dados e Laboratório de Bancos de Dados: O que são bancos de dados relacionais e structured query language. Com o volume de dados que os sistemas requerem na atualidade conhecimento de como utilizar o banco de dados é essencial.


Conhecimentos necessários para aprimorar o sistema


  • Compreender necessidades e desafios de uma área de recursos humanos de uma empresa.
  • Aprender sobre outras áreas do conhecimento para popular o sistema com mais matrizes de competência.
  • Utilizar o sistema em alguma empresa e observar os resultados obtidos.
